# VibeCast 开发计划

> 基于 PRD.md V1.0 制定。目标：Android 网页承载文本输入 + 输入法语音识别，Mac 端负责文本镜像、目标聚焦与发送。

---

## 一、技术选型（确定方案）

| 层 | 选型 | 理由 |
|---|---|---|
| Mac 服务 | **Swift 原生菜单栏 App**（方案 A） | 直接用 Accessibility API、权限清晰、长期运行稳定、开机启动简单。避免方案 B 的双进程 IPC 复杂度 |
| 内嵌 Web/WS 服务 | Swift `Network.framework`（NWListener）自建 HTTP + WebSocket | 单进程、无外部依赖、随 App 生命周期 |
| 手机前端 | **TypeScript + Vite + 原生 DOM**，无重框架 | PRD 22.1 推荐，输入法兼容性最佳 |
| 前端状态/草稿 | `localStorage`（4 个目标独立草稿 + revision） | PRD 5.4 |
| 同步协议 | WebSocket + JSON 完整快照 + 单调 revision | PRD 11/12 |
| 文本写入 | MVP 用「剪贴板全量替换 + 恢复」降级方案；二期加 AXValue 直写 | PRD 10/23 |

> Bundle ID 一律不写死，由配置/安装向导从运行中应用读取（PRD 8）。

## 二、项目结构

```
VibeCast/
├── mac/                      # Swift 菜单栏 App（SPM 或 Xcode 工程）
│   ├── Sources/VibeCast/
│   │   ├── App/              # 菜单栏 UI、状态、生命周期、开机启动
│   │   ├── Server/           # HTTP 静态托管 + WebSocket
│   │   ├── Protocol/         # 协议 Codable 模型（与前端共享语义）
│   │   ├── Session/          # 会话/Revision/目标绑定状态机
│   │   ├── Accessibility/    # 权限检测、应用激活、聚焦策略、文本写入
│   │   ├── Targets/          # 目标 Profile 配置加载/校验
│   │   └── Diagnostics/      # 脱敏日志、诊断导出
│   └── Tests/                # 协议/版本控制/状态机单元测试
├── web/                      # 前端
│   ├── src/
│   │   ├── ws/               # WebSocket 客户端、重连、快照队列
│   │   ├── ime/             # 组合输入事件处理、节流/防抖
│   │   ├── store/            # 4 目标草稿 + revision + 持久化
│   │   ├── ui/               # 卡片、状态展示、发送/清空按钮
│   │   └── main.ts
│   ├── index.html
│   └── vite.config.ts
├── shared/                   # 协议 schema 文档（前后端对齐的唯一来源）
├── docs/                     # 安装/配置/已知限制/卸载文档
└── DEVELOPMENT_PLAN.md
```

## 三、里程碑与阶段（对应 PRD 26 执行顺序）

### M0 — 项目骨架与协议定义
- [ ] 建立 mac/ 与 web/ 工程骨架、构建命令跑通
- [ ] 编写 `shared/protocol.md`：定义全部消息（hello/hello_ack/select_target/target_status/text_snapshot/text_ack/send/send_result）
- [ ] Mac、Web 双端各自实现协议 Codable / TS 类型
- **产出**：`npm run dev` 出空白页；`swift run` 启动空菜单栏

### M1 — 手机前端（可离线运行）
- [ ] 四目标卡片单列布局、当前选中高亮、底部连接状态
- [ ] 标准 `<textarea>`（非 contenteditable），支持中文/混合/长文本
- [ ] 4 个独立草稿 + 各自 revision，localStorage 持久化与刷新恢复
- [ ] 组合输入事件全套监听（composition* / beforeinput / input / selectionchange）
- [ ] 节流策略：普通输入 80–150ms 防抖，组合期 200–300ms 节流，compositionend 立即同步
- [ ] 同步状态文字化（PRD 5.5 全部状态，不只靠颜色）
- [ ] 发送/清空按钮交互与禁用逻辑（PRD 5.6/5.7）
- **验收对应**：24.3 / 24.4 / 24.7（前端部分）

### M2 — Mac 服务与连接
- [ ] HTTP 托管前端静态资源（局域网 HTTP）
- [ ] WebSocket 长连接、hello/hello_ack 握手
- [ ] 配对令牌（MVP：Mac 生成令牌 + 菜单栏展示，二期上二维码）
- [ ] 仅监听本机 + 局域网，不暴露公网（PRD 15.1）
- [ ] 菜单栏 UI：运行状态、局域网地址、连接数、当前目标、权限状态、重启/日志/退出
- **验收对应**：连接建立、状态展示

### M3 — 应用激活与聚焦
- [ ] 辅助功能权限检测，无权限时禁止注入/发送并提示（PRD 7.2）
- [ ] 通过 Bundle ID 激活应用（已运行/启动/等待窗口/超时错误）
- [ ] 聚焦策略：**策略一 应用快捷键**（MVP 主路径）+ 策略三 恢复上次焦点
- [ ] 目标元素绑定记录（PID/BundleID/窗口/会话 ID）与有效性校验
- [ ] 失焦检测 → 暂停同步、不盲写、手机显示「目标已失焦」
- **验收对应**：24.1 / 24.8 / 16.2 / 16.3

### M4 — 文本镜像（核心安全路径）
- [ ] 剪贴板全量替换降级方案：聚焦校验 → 备份剪贴板 → 全选 → 写入 → 粘贴 → 验证 → 恢复剪贴板
- [ ] **严禁**未验证目标焦点就 Cmd+A；严禁逐字模拟键盘（PRD 10.3）
- [ ] Revision 单调校验：只应用更高版本，旧包丢弃（PRD 12.1）
- [ ] text_snapshot → text_ack 全链路
- **验收对应**：24.1 / 24.3 / 24.4 / 24.10（剪贴板保护）

### M5 — 两阶段发送
- [ ] send 携带 revision；未应用则先应用，仅有低版本则拒绝发送（PRD 11.4）
- [ ] 发送前重新校验目标焦点与文本
- [ ] 可配置发送动作（Enter / Cmd+Enter / 自定义）
- [ ] 发送幂等（同 session+target+revision 不重复提交，PRD 16.6）
- [ ] send_result 成功/失败回传，失败不自动重试
- **验收对应**：24.5 / 24.6

### M6 — 健壮性与错误处理
- [ ] 断网：保留草稿、禁发、自动重连、重连后只同步最新完整快照、不自动发送（PRD 16.4）
- [ ] Mac 睡眠/唤醒重连后需重新选目标（PRD 16.5）
- [ ] 单一活动会话约束（PRD 12.2）
- [ ] 脱敏日志（只记长度/哈希，不记全文/令牌/剪贴板，PRD 21）
- **验收对应**：24.9

### M7 — 配置、开机启动、目标适配
- [ ] 目标 Profile 配置（YAML/JSON，PRD 8）与校验
- [ ] Mac 配置页面：每目标全部可编辑项 + 「测试目标」按钮（PRD 20）
- [ ] 登录后开机自动启动（ServiceManagement / SMAppService）
- [ ] 四应用实测适配：Codex / WorkBuddy / Notion / CodeBuddy
- [ ] **Notion 特殊处理**：MVP 限定为 AI 对话模式或当前文本块模式，禁止整页全选替换（PRD 14）

### M8 — 测试、联调、交付
- [ ] 单元测试：协议序列化、Revision 并发、状态机、发送幂等（PRD 25/24 自动化部分）
- [ ] 真机联调：Android Chrome + 微信输入法（验收基线环境，PRD 18）
- [ ] 走完 24.1–24.10 全部验收用例
- [ ] 文档：安装 / 启动停止 / 权限说明 / 目标配置 / 已知限制 / 卸载 / 构建命令
- [ ] 产出可分发 App（菜单栏应用打包）

## 四、关键风险与对策

| 风险 | 对策 |
|---|---|
| 微信输入法语音模式无法网页自动唤起（PRD 已知限制 1） | 接受为产品边界，首次使用文字提示用户手动点语音按钮；不做假录音 UI |
| 组合输入事件各输入法表现不一（限制 2） | M1 真机测试为准，节流参数可配置 |
| Notion 误覆盖整页文档（限制 4） | M7 强制限定输入目标，默认禁全选替换 |
| Electron/复杂编辑器 AX 支持不全（限制 3） | MVP 用剪贴板粘贴方案兜底，二期再做 AXValue 直写 |
| 手机浏览器后台 WS 被挂起（限制 7） | MVP 仅保证前台场景，重连补最新快照 |

## 五、MVP 边界（PRD 23 第一阶段，22 项全部纳入）

聚焦：四卡片 + 四独立草稿 + 输入法输入 + WS 实时全量快照同步 + Revision + 应用激活 + 快捷键聚焦 + 剪贴板替换/恢复 + 自定义发送 + 两阶段确认 + 配对令牌 + 本地草稿 + 权限检测 + 错误提示 + 开机启动 + 脱敏日志。

二期（不在本计划首轮）：AXValue 直写、自动查找控件、Notion 深度适配、多步聚焦宏、HTTPS+PWA、二维码配对、多 Mac、发送历史等。

---

## 六、硬约束（来自 PRD 26，不可违反）

1. 不在 Mac 做语音识别，不接收/传输/保存音频
2. 网页不申请麦克风权限
3. 不把用户语音或文本发给外部服务
4. 未验证目标焦点，绝不 Cmd+A / Cmd+V 盲写
5. 不逐字模拟键盘做同步
6. 日志不记录完整用户文本
