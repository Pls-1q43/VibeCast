# VibeCast 0.1.0

VibeCast 0.1.0 is the first public release.

## Highlights

- Mobile-first prompt input from Android Chrome to macOS target apps.
- Real-time draft sync over local WebSocket with revision acknowledgement before sending.
- Configurable target apps, including built-in presets and custom Bundle IDs.
- Accessibility-driven focus, text replacement, send actions, and clear-after-send controls.
- Pairing token protection, active-client ownership, rate limiting, and redacted diagnostics.
- Sparkle update support with signed appcast entries.

## Installation Notes

- This build is ad-hoc signed and not notarized with an Apple Developer ID.
- On first launch, macOS may block the app. Open it from System Settings > Privacy & Security.
- VibeCast requires macOS Accessibility permission before it can focus apps, write text, or send.
- Mac and Android Chrome must be on the same local network.

## Known Limits

- Android voice input still needs to be started manually from the keyboard.
- The app is intended for local-network use and does not provide cloud relay.
- App-specific focus and send behavior can vary; use the configuration page to test each target.
