# VibeCast 0.3.0

VibeCast 0.3.0 adds Intel Mac support and display themes for lower-light and E-Ink-style phone use.

## Highlights

- Built the macOS app as a universal binary for both Apple Silicon and Intel Macs.
- Kept Sparkle updates on the existing feed and signing key so users coming from 0.1.x or 0.2.x can update through the same path.
- Added automatic light/dark styling for the phone input page and configuration page.
- Added a phone-side E-Ink theme with monochrome, high-contrast grayscale, and lineframe-style cards.

## Installation Notes

- This build is ad-hoc signed and not notarized with an Apple Developer ID.
- If macOS blocks the app after download, open it from System Settings > Privacy & Security.
- VibeCast requires macOS Accessibility permission before it can focus apps, write text, or send.
- Existing target configuration and phone drafts are preserved.
- Mac and Android Chrome must be on the same local network.

## Known Limits

- Android voice input still needs to be started manually from the keyboard.
- E-Ink mode is a display theme only; it does not change sync, sending, or input behavior.
- App-specific focus and send behavior can vary; use the configuration page to test each target.
