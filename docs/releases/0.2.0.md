# VibeCast 0.2.0

VibeCast 0.2.0 focuses on safer writing into document editors and clearer setup guidance for real-world target apps.

## Highlights

- Added Editor Mode for document-style apps such as Obsidian, where VibeCast should update only the text inserted during the current input round instead of replacing the whole editor.
- Added an Obsidian preset with sync-only behavior, editor-mode defaults, and a built-in target icon.
- Improved editor-mode writing so Obsidian can fall back to undoing the previous VibeCast paste and pasting the latest draft when CodeMirror does not expose an accessibility selection range.
- Prevented the configuration page from stealing the active phone input session, so testing and configuration actions no longer interrupt the current controller.
- Expanded the README and configuration docs with editor mode, dialog mode, screenshots, and setup notes.

## Installation Notes

- This build is ad-hoc signed and not notarized with an Apple Developer ID.
- If macOS blocks the app after download, open it from System Settings > Privacy & Security.
- VibeCast requires macOS Accessibility permission before it can focus apps, write text, or send.
- Existing target configuration is preserved. Review Obsidian in the configuration page if you previously customized its sync or write mode.
- Mac and Android Chrome must be on the same local network.

## Known Limits

- Obsidian's fallback uses Undo to replace the previous VibeCast insertion when accessibility selection APIs are unavailable. Avoid editing inside that inserted segment from the Mac while the same phone input round is still syncing.
- Editor Mode is designed for the current input round; it does not perform arbitrary document diffing or multi-cursor edits.
- Android voice input still needs to be started manually from the keyboard.
- App-specific focus and send behavior can vary; use the configuration page to test each target.
