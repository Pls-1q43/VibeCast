# 已知限制

1. 网页点击文本框只能唤起 Android 输入法，**不能自动进入微信输入法的语音模式**，需用户手动点语音按钮。
2. 不同版本微信输入法对组合输入事件的表现可能不同，需真机测试。
3. Electron 应用或复杂编辑器对 macOS Accessibility 的支持可能不完整；
   此时 AXValue 直写可能失败，会降级为剪贴板粘贴（若该目标允许全选替换）。
4. **Notion 普通页面不适合直接全选替换**，必须限定输入目标（AI 对话框或当前文本块），
   默认 `allowSelectAllReplace=false`。
5. 目标应用更新后，快捷键或 Accessibility 结构可能变化，需重新配置/测试。
6. 手机端是同步期间的唯一事实源，Mac 端手动修改可能被下一次手机同步覆盖。
7. 手机浏览器进入后台后，WebSocket 可能被 Android 暂停；本版本主要面向前台主动操作场景。
8. 若需长期后台连接，未来可能需要 Android 原生壳应用。
9. 局域网地址可能变化；可通过复制访问地址、固定主机名或后续二维码/mDNS 改善访问体验。
10. 配对采用 URL 内长期令牌（MVP）；二维码一次性配对码、HTTPS/PWA 为二期增强。

## 安全边界（设计约束，非缺陷）

- 不在 Mac 做语音识别，不接收/传输/保存音频。
- 网页不申请麦克风权限。
- 不把语音或文本发送给任何外部服务，全部局域网内传输。
- 未验证目标焦点绝不写入文本；未授权辅助功能时禁止注入与发送。
- 日志不记录完整文本、令牌、剪贴板内容（仅记长度与哈希）。
