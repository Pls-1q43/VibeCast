# 能力边界与最佳实践

VibeCast 的核心体验是把手机输入法产生的文本稳定镜像到 Mac 目标应用。理解下面这些边界，可以让输入更快、更稳、更可控。

## 输入法与手机浏览器

1. 网页通过标准文本框唤起 Android 输入法，语音按钮由输入法自己提供。
2. 不同输入法和 Android 版本的组合输入事件表现可能不同，建议用你的主力手机和输入法做一次真实输入测试。
3. 手机浏览器进入后台后，WebSocket 可能被系统挂起；前台输入体验最稳定。
4. 本地 HTTP 页面会按浏览器安全模型运行，VibeCast 已为 UUID 等能力提供兼容兜底。

## macOS 与目标应用

1. 辅助功能权限是激活应用、聚焦输入框、写入文本和发送动作的基础能力。
2. Electron、WebView、contenteditable 或复杂编辑器的 Accessibility 支持差异较大，配置页的“测试”按钮是最可靠的验收方式。
3. AXValue 直写失败时，只有目标配置允许，VibeCast 才会降级为剪贴板写入。
4. 目标应用更新后，快捷键、窗口结构或输入框行为可能变化，重新测试目标配置即可恢复稳定体验。
5. 目标绑定来自明确的选择和聚焦流程，避免文本写入到意外位置。

## Notion 最佳实践

Notion AI 输入框和普通文档块的行为不同：

- Notion AI 对话框：确认焦点在 AI 输入框内，可使用 `clipboard_replace`。
- 当前文本块：先手动点好文本块，使用 `preserve_last_focus`，并将发送策略设为 `none` 或使用光标插入。

对普通页面保持 `allowSelectAllReplace=false`，可以避免整页内容被错误替换。

## 隐私与控制

1. 语音识别发生在 Android 输入法内部。
2. 网页不请求麦克风权限。
3. VibeCast 不接收、传输或保存音频。
4. Mac 端不把用户文本发送给外部服务。
5. 配对令牌保护手机页面与 Mac 服务的连接。
6. 诊断日志不记录完整文本、令牌或剪贴板内容。

更多细节见 [安全与隐私](SECURITY.md)。
