# 已知限制

VibeCast 的 v1 目标是“同一局域网内，单用户，用 Android 输入法把文本安全镜像到 Mac 目标应用”。下面这些限制属于当前设计边界或 MVP 取舍。

0.1 按“通用可配置目标”发布，不承诺 Codex、Notion、WorkBuddy、CodeBuddy 等第三方应用特定版本适配。发布包为本地签名 `.app`，不做 Apple Developer ID 公证。

## 输入法与手机浏览器

1. 网页点击文本框只能唤起 Android 输入法，不能自动进入微信输入法语音模式。
2. 不同输入法和 Android 版本的组合输入事件表现可能不同，需要真机验证。
3. 手机浏览器进入后台后，WebSocket 可能被系统挂起；当前主要面向前台主动输入场景。
4. 局域网 HTTP 页面不是安全上下文，部分浏览器 API 需要兜底实现；项目已为 UUID 生成做了降级。

## macOS 与目标应用

1. 必须授予辅助功能权限，否则无法激活应用、聚焦输入框、写入文本或发送。
2. Electron、WebView、contenteditable 或复杂编辑器的 Accessibility 支持可能不完整。
3. AXValue 直写失败时，只有目标配置允许时才会降级为剪贴板写入。
4. 目标应用更新后，快捷键、窗口结构或输入框行为可能变化，需要重新测试目标配置。
5. VibeCast 不会自动识别任意 App 的任意输入框，目标绑定必须来自明确的选择和聚焦流程。

## Notion 特别说明

Notion 普通页面不适合随意全选替换。推荐只在两类场景使用：

- Notion AI 对话框：确认焦点在 AI 输入框内，使用 `clipboard_replace`。
- 当前文本块：先手动点好文本块，使用 `preserve_last_focus`，并将发送策略设为 `none`。

默认不要为 Notion 页面开启会导致整页内容被全选替换的配置。
0.1 的 Notion 默认值面向 AI 输入框，而不是普通页面文本块；如果要写普通页面文本块，请改回 `sendMode=none` 并关闭 `allowSelectAllReplace`。

## 安全边界

1. 不在 Mac 做语音识别。
2. 网页不请求麦克风权限。
3. 不接收、传输或保存音频。
4. 不把用户文本发送给外部服务。
5. 当前配对采用 URL 内长期令牌；请只在可信局域网使用。
6. 诊断日志不记录完整文本、令牌或剪贴板内容。

更多细节见 [安全与隐私](SECURITY.md)。
